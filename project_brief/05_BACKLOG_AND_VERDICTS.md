# 05 · BACKLOG ІДЕЙ + БРУТАЛЬНІ ВЕРДИКТИ

> Ідеї, накидані на етапі планування. Капчу, щоб не загубились, але **не будуємо все одразу**.
> Принцип: спершу ОДИН сайт, що конвертить → потім розширення по черзі. `1+1=11`.

## A. Дві версії сайту з перемикачем (реф-конкуренти vs наші покращення)
- **Інстинкт — правильний** (порівняння = сильна продажна психологія: клієнт відчуває контроль і сам обирає кращу).
- **Пастка:** дві ПОВНІ продакшн-версії = 2× верстка, 2× баги, 2× підтримка, розмитий фокус. А сайти-конкуренти вже існують онлайн — Віктор їх і так відкриє в сусідній вкладці.
- **ВЕРДИКТ → RESHAPE:** будуємо **ОДНУ** сильну (нашу покращену) версію. Порівняння даємо на рівні **мокапів/скриншотів** (дешево), не двох робочих сайтів. Якщо дуже треба тогл — це theme-switch однієї кодової бази, не два сайти.

## B. OpenAI Codex пише код + генерує «повністю реалістичні» картинки
- **Код → НІ.** Ми В Claude Code. Я пишу сайт напряму. Додавати Codex = зайвий tooling-спрол + координація двох систем заради того, що вже вміємо.
- **AI-картинки дерев → НАЙРИЗИКОВІШЕ МІСЦЕ.** Це сайт МАЙСТРА дерев для клієнтів-знавців. Наш власний урок `FORGE_LOCKS #015`: FLUX = мутанти на складних суб'єктах. Хвоя/крони/клени — максимально складні. Бонсай-експерт спалить фейк за секунду → удар по довірі (а довіра = весь сенс сайту).
- **ВЕРДИКТ → AI лише як ЯВНИЙ плейсхолдер** (сірі блоки/нейтральні стоки), щоб тримати макет, поки Віктор не вставить РЕАЛЬНІ фото (6 before/after + 3 робочих — це його актив). «Реалістичний обман» на цьому сайті заборонений.

## C. 1-клік голосовий intake клієнта → транскрипція → напряму до Віктора
- **ЗОЛОТО. Це `1+1=11`** — переюзує STT-пайплайн (faster-whisper у WSL), який ми щойно підняли для його голосових. На-бренді: розширює «надішли фото → консультація» на «розкажи голосом».
- **АЛЕ:** це **v2-фіча**, не v1. Спершу ядро має конвертити (фото→WhatsApp, дзвінок, ціни).
- **ВЕРДИКТ → PARK (пріоритет 1 у v2).** Будуємо після того, як базовий сайт працює і приймає ліди.

## D. Дизайн-плагін «godmode» пак (frontend-design, playground, skills тощо)
- **Концепт корисний** (production-grade дизайн замість generic-AI вигляду).
- **Перевір факти:** список у меседжі виглядає AI-згенерованим (цитати [1]–[7]) — назви репо/плагінів **звір перед масовим install**, частина може бути неточною. `/plugin install` — інтерактивна команда, я її в цій сесії **не запускаю**.
- **ВЕРДИКТ → пізніше, точково.** Більше тулзів ≠ кращий сайт. Один-два перевірені (напр. frontend-design концепт) — коли дійдемо до верстки, не зараз.

## E. ⭐ VOICE → BLOG ENGINE (1+1=11, найбільший резонанс — на підтвердження)
- **Ідея:** ми вже ПОБУДУВАЛИ STT-пайплайн (faster-whisper, WSL). Віктор ОБОЖНЮЄ говорити (надиктував 2.4 год). Блог потребує контенту вічно. З'єднати: **Віктор шле голосові → транскрипція → AI робить німецьку статтю в його голосі → SEO росте сам.**
- **Що множить:** SEO (довга гра) × контент × автентичність × актив, який уже є (STT) × recurring-цінність Андрія (він гонить пайплайн).
- **MVP (мале):** простий повторюваний workflow, не CMS-автоматизація. Голосове → готова чернетка статті за хвилини.
- **ВЕРДИКТ → ✅ ПІДТВЕРДЖЕНО:** не блокує збірку; вмикаємо після першого сайту як двигун контенту. Живе також у 5-й вершинці (Persönliche Baum-Vision, [13](13_COPY_DECK_DE.md) §5★) як «голосовий діагноз».
