## Технічні вимоги (для Codex)
- Mobile-first; має **відчуватися швидким** на реальному телефоні (без фанатичної гонитви за балами — див. [10_PRINCIPLES](10_PRINCIPLES.md)).
- DE (i18n-готовність під EN — апсейл).
- Семантичний HTML, sticky mobile CTA, lazy-load фото, alt-тексти (SEO).
- Розділ AI-плейсхолдерів: чітко позначені тимчасові зображення → Віктор замінює на реальні.

## Codex build-package (що я підготую, коли скелет+контент+відповіді готові)
1. `CONTEXT.md` — повний контекст проєкту (хто Віктор, меседж, аудиторія, тон).
2. `DESIGN_TOKENS.md` — кольори/типографіка/spacing (з robert+caesar напряму).
3. `COPY_DECK_DE.md` — фінальні тексти DE по блоках.
4. `BUILD_SPEC.md` — структура файлів + покрокові інструкції збірки.
5. `IMAGE_BRIEF.md` — список потрібних зображень + промпти (для image-моделі, НЕ Codex — див. нижче) + які заміняє Віктор.
6. `PROMPTS.md` — готові промпти для Codex по кожному кроку.

> ⚠️ **Codex генерує КОД, не картинки.** Фотореалістичні плейсхолдери робить **image-модель** (напр. gpt-image-1 / Midjourney), Codex лише вставляє їх у верстку. Деталі — у відповіді в чаті.
