# 10 · ПРИНЦИП СТВОРЕННЯ САЙТУ (керівна доктрина)

> Це governing-доктрина. Кожне рішення в дизайні/збірці звіряється з нею. Codex і я зобов'язані їй слідувати.

## Суть
Максимально якісний сайт у межах бюджету/часу: **красивий + швидкий + довірчий + з контентом + продаючий**.
Критичне обмеження: **НЕ можна, щоб сайт погано/довго вантажився з телефона.** Але **без фанатичної ультраоптимізації заради сухих балів** — сайт має *відчуватися* швидким, легким, приємним на мобільному.

## Головний баланс
- НЕ робити порожнім «заради швидкості».
- НЕ перевантажувати ефектами/важким відео/анімаціями/зайвими бібліотеками «заради краси».
- Баланс: **якість + швидкість + довіра + контент + продаж.**

## 🚦 ПРАВИЛО ADD/CUT (головний фільтр будь-якого елемента)
**ДОДАЄМО**, якщо елемент:
- робить красивіше **АБО** посилює довіру **АБО** допомагає продажу,
- **І** не вбиває мобільну швидкість.

**ВИРІЗАЄМО без жалю**, якщо елемент просто «прикольний», але важкий/повільний АБО не допомагає клієнту прийняти рішення.

## Операціоналізація (як «відчувається швидким» виглядає в коді — НЕ фанатично)

### Зображення (головна вага на мобілці)
- Формат **AVIF/WebP** (фолбек JPEG); ніколи не сирий PNG/повнорозмірний JPEG у героях.
- **srcset/sizes** — телефону віддаємо мобільний розмір, не десктопний.
- **lazy-load** усе нижче згину (`loading="lazy"`); герой — eager + `preload`.
- Орієнтир ваги: контент-фото ~100–200 KB, герой ≤ ~300 KB.
- Завжди `width/height` (проти стрибків layout / CLS).

### Шрифти
- Максимум **1 Google-шрифт** (1–2 накреслення) **або** системні. `font-display: swap`. Preload того, що в героях. Subset latin.

### Анімації
- Тільки **CSS** (`transform`/`opacity`), **без** GSAP/AOS та інших бібліотек анімації.
- Поважати `prefers-reduced-motion`. Делікатно, не миготіння.

### Відео
- **Без** важкого autoplay-відео в героях. Якщо відео — коротке, стиснене, muted-loop із `poster`, або click-to-play з постером. Для v1 краще взагалі без відео.

### Скрипти / бібліотеки
- Статичний HTML/CSS. **Без** jQuery/Bootstrap/React для маркетингового сайту.
- Ванільний JS, мінімум. Галерея — CSS `scroll-snap`, не важкий слайдер-плагін.

### Архітектура
- Multipage = окремі **легкі HTML-сторінки**, не важкий SPA. Кожна сторінка вантажить лише своє.
- Хостинг Vercel (швидка віддача) → потім .ch.

### Sanity-бюджет (орієнтир ВІДЧУТТЯ, не марнославні бали)
- Герой/LCP-зображення оптимізоване + preload → головний контент видно швидко на мобільному 4G.
- Орієнтир ваги сторінки на мобілці: **~< 1–1.5 MB** разом із фото.
- Жодних render-blocking важких ресурсів.
- **Головна перевірка — відкрити з реального телефона** (як сказав Андрій), а не ганятись за Lighthouse-100.
